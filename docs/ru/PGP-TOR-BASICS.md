# PGP + Tor — базовая шпаргалка (RU)

## Зачем два слоя

| Инструмент | Что защищает |
|------------|--------------|
| **Tor** | Куда ты ходишь (IP, маршрут, .onion) |
| **PGP** | Что ты пишешь (текст, файлы) |

Tor **не** шифрует содержимое сообщений на форумах/в чатах — для этого PGP.

---

## Минимальный старт на этой станции

```powershell
Install-PgpToolkit.ps1      # установить GnuPG
Configure-PgpIdentity.ps1   # создать ключ (или: pgp-setup)
pgp-export                  # публичный ключ для контактов
pgp-fingerprint             # отпечаток для проверки
```

---

## Три вещи, которые нужно понять

### 1. Публичный ключ
- Можно давать всем
- Контакт шифрует сообщение **для тебя**
- Файл: `C:\Security\pgp\public-key-*.asc`

### 2. Приватный ключ + passphrase
- **Никому не показывать**
- Без passphrase ключ бесполезен даже у тебя
- Хранится в `%APPDATA%\gnupg\` (GnuPG)

### 3. Fingerprint (отпечаток)
- Длинная строка SHA — «паспорт» ключа
- Сверяешь с контактом **другим каналом** (не в том же чате, где met)
- Защита от подмены (MITM)

---

## Tor — базовая гигиена

1. **Tor Browser** — отдельный браузер, не смешивай с обычным профилем
2. **Не логинься** в личные аккаунты (Google, VK) из Tor Browser
3. **Не качай** и не открывай файлы из непроверенных источников
4. **JavaScript** — в Tor Browser оставь уровень безопасности по умолчанию или выше
5. **Псевдоним** — отдельный от реальной жизни; PGP ключ с реальным именем = deanonymization

---

## Как выглядит переписка с PGP

**Отправить зашифрованное сообщение (концепт):**
1. Получить **публичный ключ** контакта (.asc)
2. Импорт: `gpg --import contact.asc`
3. Проверить fingerprint: `gpg --fingerprint`
4. Шифровать текст/файл для его key id
5. Отправить ciphertext в чат (Tor)

**Получить:**
1. `gpg --decrypt message.asc` (или через Kleopatra GUI)
2. Ввести **свою** passphrase

## Backup (обязательно)

| Что | Где |
|-----|-----|
| Revocation certificate | `C:\Backups\Workstation\pgp\` |
| Экспорт публичного ключа | `C:\Security\pgp\` |
| Приватный ключ | backup всего `%APPDATA%\gnupg\` на **зашифрованный** носитель |

Revocation — «отозвать» ключ если passphrase утекла.

## GUI (если CLI сложно)

После установки GnuPG есть **Kleopatra** (Пуск → Kleopatra):
- Создать ключ
- Import / Export
- Encrypt / Decrypt файлов

---

## Чего PGP НЕ делает

- Не анонимизирует сам по себе (нужен Tor + отдельная идентичность)
- Не защищает от malware на ПК
- Не скрывает метаданные (когда писал, размер) — только содержимое
- Не заменяет здравый смысл и законы

## Команды HOME BASE

| Команда | Действие |
|---------|----------|
| `pgp-help` | эта шпаргалка в терминале |
| `pgp-setup` | создать ключ |
| `pgp-status` | список ключей |
| `pgp-export` | экспорт public .asc |
| `pgp-fingerprint` | отпечаток |
| `pgp-encrypt -To KEYID -File path` | зашифровать файл |
| `pgp-decrypt -File path.gpg` | расшифровать |

## Дальше (когда освоишь базу)

- Отдельный ключ **только** для Tor-идентичности (не смешивать с рабочим email)
- **Tails** USB — для максимальной изоляции (опционально)
- `gpg --verify` — проверка подписи файлов/сообщений
